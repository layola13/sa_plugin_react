# React Plugin

`sa_plugin_react` is a React-on-SAX plugin for SA. It accepts `.sax` component files, normalizes React DOM style tags, attributes, and event names, then emits the same browser-ready bundle shape as SAX.

## What `sa react build` produces

For an input such as `demos/react_counter.sax`, the build command generates:

- `app.wasm`
- `airlock.js`
- `index.html`
- `app.sa`

## Example

```bash
SA_PLUGINS_PATH=$PWD/zig-out/lib/libreact.so /home/vscode/projects/sci/zig-out/bin/sa react build demos/react_counter.sax --out-dir /tmp/react-counter
```

Additional component source files can be composed into the same SAX program with `--include` or `-I`:

```bash
SA_PLUGINS_PATH=$PWD/zig-out/lib/libreact.so /home/vscode/projects/sci/zig-out/bin/sa react check app.sax --include lib/components.sax
```

Includes are resolved first relative to the root `.sax` file. If that file is not present, React also searches `SA_REACT_INCLUDE_PATH`, share directories inferred from `SA_PLUGINS_PATH`, installed plugin share directories under `$SA_PLUGINS_HOME/installed/<plugin>/current/share`, and the local development plugin workspace. This allows a plugin that only installs SAX assets, for example `mui/material.sax`, to be consumed without adding a plugin-specific source expander:

```bash
SA_PLUGINS_PATH=$PWD/zig-out/lib/libreact.so:/home/vscode/projects/sa_plugins/sa_plugin_mui/zig-out/lib/libmui.so /home/vscode/projects/sci/zig-out/bin/sa react check demos/mui_all_components_from_library.sax --include mui/material.sax
```

## Notes

- This is a compatibility layer on top of the existing SAX lowering pipeline, not a full React Fiber reimplementation.
- The generated HTML keeps the same CSP and DOM airlock boundary as SAX.
- General JavaScript computed object key expressions beyond the current scalar arithmetic/state/string-ternary subset, arbitrary JavaScript object spread expressions beyond the current ptr-state and simple boolean-ternary ptr-state subset, and complex non-scalar object/string expressions beyond the current ptr-state, numeric/boolean-state, and simple boolean-ternary state subset are still follow-up runtime layers.
- Full JavaScript object prop ABI is not complete yet; `select multiple` currently accepts comma/newline-delimited strings as the SA-compatible representation.
- Full React component instance ref semantics are not complete yet; current `ref` support stores DOM handles into `i64` state slots, mounted child component context pointers into `ptr` state slots, and invokes callback refs through the SAX ref callback ABI.
- Full keyed reconciliation is not complete yet; current `key` support preserves metadata and prevents accidental DOM `key` attributes.
